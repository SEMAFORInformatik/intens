\subsubsection{Navigator}
\label{sec:uinavigator}

tree using folders and subfolders. \\

in section \nameref{sec:uinavigatorPixmaps} page \pageref{sec:uinavigatorPixmaps}. \\

\vspace{2ex}
By default, the \NAVIGATOR{} shows a header node (using the structure's \LABEL{}) to group its entries,
and shows all nodes open. Alternative behaviour can be configured using the following options. \\

Level refers to the depth of a nested variable:
\begin{itemize}
\item parent is level 1
\item parent.elements is level 2
\item parent.elements[n].name is level 3
\end{itemize}

\begin{tabularx}{\textwidth}{l|X}
Root list options & description \\
\hline
\AUTOLEVEL        & Hides the header node of structs with a single entry. \\
\HIDEEMPTYFOLDERS & Does not display header nodes without entries. \\
\OPENLEVELS       & Opens (expands) nodes up to and including tree-level 'integer'.
                    Closes (collapses) nodes after level 'integer'. \\
\FIRSTLEVEL       & Does not display header nodes before level 'integer'. \newline
                    This option is required for \DIAGRAM{} to specify the level
                    to render. \\
\LASTLEVEL        & Hides nodes after level 'integer'. \\
\end{tabularx}

Example of functions to handle events like {\bfseries select} or {\bfseries drag and drop}:

\begin{boxedminipage}[t]{\linewidth}
\begin{intens}[escapechar=@]
DESCRIPTION "Implementing menu and drag'n'drop functions";
DATAPOOL
  STRUCT Motor {
    INTEGER   id;
    REAL      nom_power        {TAG=Tpower};
    STRING    id_manuf         {TAG=Tmanuf};
    STRING    model            {TAG=Tname};
    ...
  };
  Motor motors  {FUNC=@\hl{select\_motor}@};
END DATAPOOL;
FUNCTIONS
  FUNC INIT {
    motors[0].nom_power = 2.5;
    motors[0].id_manuf = "ABB";
    motors[0].model="QU 132";

    motors[1].nom_power = 7.5;
    motors[0].id_manuf = "Siemens";
    motors[0].model="1PH8131";
    ...
  };
  FUNC @\hl{select\_motor}@ {
    IF( REASON_SELECT ){
      SET_MSG( THIS.model, " selected" ); }
    ELSE IF( REASON_UNSELECT ){
      SET_MSG( THIS.model, " unselected" ); }
    ELSE IF( REASON_ACTIVATE ){
      SET_MSG( THIS.model, " activated" ); }
    ELSE IF( REASON_DROP ){
      SET_MSG( SOURCE.model, " dropping to ", THIS.model ); }
  };
END FUNCTIONS;
UI_MANAGER
  NAVIGATOR
    motornav { COL( "" Tname:20
                  , "Manuf." Tmanuf:12
                  , "Output [kW]" Tpower:15 )} (
      motors[*] = "Motors" );
END UI_MANAGER;
\end{intens}
\end{boxedminipage}

\subsubsection{Navigator IconView}
\label{sec:uinavigatorIconView}
This \NAVIGATOR{} type is used to show several elements
that can be dragged to a \DIAGRAM.

The \NAVIGATOR{} itself has one \COL{} with a \TAG{}.
The data item (parent in the example below) must have an attribute with that tag
(name in the example).
The \ICONVIEW{} shows all children of it that have an attribute with that tag
(parent.elements[*] with name in the example).

\begin{boxedminipage}[c]{0.62\linewidth}
\begin{intens}
DATAPOOL
  STRUCT
    Elements {
      STRING {EDITABLE}
        name { TAG=(nametag) }, type; };
  STRUCT
    Parent {
      STRING {EDITABLE}
        name { TAG=(nametag) };
      Elements elements; };
  Parent parent;
END DATAPOOL;

UI_MANAGER
  NAVIGATOR { ICONVIEW }
    iconView_navigator{
      SIZE(200, 200), COL ("" nametag)}(
      parent { AUTOLEVEL });
  FORM
    iconView_form { HIDE_CYCLE } (
      iconView_navigator );
END UI_MANAGER;

FUNCTIONS
  FUNC
    INIT {
      parent.elements[0].name = "A";
      parent.elements[1].name = "B";
      ...

      parent.elements[0].type = "A";
      parent.elements[1].type = "B";
      ...
    };
END FUNCTIONS;
\end{intens}
\end{boxedminipage}

\subsubsection{Navigator Pixmaps}
\label{sec:uinavigatorPixmaps}
The navigator uses pixmaps to represent elements. This section explains the
process used to resolve which pixmaps are displayed.

The routine consists of five sequential steps.
For each step, the program tries the first possibility against all subsequent steps.
If no match is found, it tries the second possibility, and so on.
The process stops as soon as a valid pixmap is located.

\begin{itemize}
\item Determine Identifier
  \begin{itemize}
  \item Value of \STRING{} attribute 'node\_name' ( \DIAGRAM{} only ) \newline
  This is only evaluated if the \STRING{} attribute 'type' also has a value.
  If 'node\_name' is used, the next two steps are skipped,
  and the program proceeds directly to searching for file extensions.
  \item Value of \STRING{} attribute 'type' ( \DIAGRAM{} and \ICONVIEW{} only )
  \item Variable name of the element list: \newline
  (e.g., 'elements' in the \ICONVIEW{} example)
  \end{itemize}

\item Determine pixmapName
  \begin{itemize}
  \item Search the resource file section ([Navigator], [Diagram] or [IconView])
        for an entry: identifier.iconPixmap; \newline
        If no entry is found, pixmapName defaults to the identifier.
  \end{itemize}

\item Search for a pixmap with the following name patterns:
  \begin{itemize}
  \item pixmapName
  \item lower(identifier) (all lower case)
  \item lower(identifier)-small
  \item 'default' (\DIAGRAM{} only)
  \end{itemize}

\item Search for the following pixmap types (by extension):
  \begin{itemize}
  \item *.xpm
  \item *.bmp
%gentoo  \item *.gif
%gentoo  \item *.mng
  \item *.jpg
%  \item *.jpeg
  \item *.pbm
  \item *.pgm
  \item *.png
  \item *.pnm
  \item *.ppm
  \item *.tif
  \item *.svg
  \item *.eps
%gentoo  \item *.xbm
  \end{itemize}

\item Search the following directories (in order):
  \begin{itemize}
  \item Directories defined in {\bfseries BITMAP\_PATH} environment variable
  (separated by ':' on Linux or ';' on Windows).
  \item {\bfseries APPHOME}/bitmaps
  (where {\bfseries APPHOME} is an environment variable).
  \item {\bfseries IntensHome}/bitmaps
  ({\bfseries IntensHome} is the parent directory of the current \INTENS{} executable used).
  \item ./bitmaps (local subdirectory).
  \item . (current working directory).
  \end{itemize}

\end{itemize}
